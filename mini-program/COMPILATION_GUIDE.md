## 微信小程序编译问题

### 现象
运行 `pnpm dev:mp-weixin` 时出现大量 antd 相关警告，编译时间很长（2-5 分钟）。

### 原因
1. **antd 包很大** - antd 是一个完整的 UI 库，包含大量组件
2. **依赖扫描** - 即使未实际使用，Vite 也会扫描和检查依赖树
3. **平台适配** - 小程序环境需要额外的时间处理平台差异

### 影响评估
- ✅ **不影响功能** - 这些只是警告，不会阻止编译
- ✅ **编译会完成** - 耐心等待即可完成编译
- ⚠️ **初次较慢** - 后续增量编译会快很多

## 推荐开发方案

### 方案一：使用 H5 端开发（推荐）

**为什么推荐：**
- ✅ 与微信小程序使用完全相同的代码
- ✅ 支持 Chrome DevTools，调试方便
- ✅ 热更新速度快（约 1 秒）
- ✅ 可以实时查看效果

**启动方式：**
```bash
# 1. 启动后端服务
pnpm dev

# 2. 新终端窗口，启动小程序 H5 端
cd mini-program && pnpm dev:h5

# 3. 访问 http://localhost:3607/
```

### 方案二：优化微信小程序编译

如果确实需要使用微信开发者工具实时预览，可以：

#### 1. 配置 Vite 排除 antd

在 `mini-program/vite.config.ts` 中添加：
```typescript
build: {
  rollupOptions: {
    external: ['antd', 'antd/es/*', '@ant-design/icons'],
  },
},
```

#### 2. 使用生产构建

```bash
cd mini-program && pnpm build:mp-weixin
```
生产构建会更快且更优化。

#### 3. 后台编译+预览

```bash
# 后台启动编译，先去做其他事情
cd mini-program && pnpm dev:mp-weixin &

# 等待 5-10 分钟后，用微信开发者工具打开 dist/dev/mp-weixin
```
